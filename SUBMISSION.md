# Submission Packet

This repository is a Solana smart transaction infrastructure stack for the Superteam Nigeria Advanced Infrastructure Challenge. It is built around Jito bundles, Yellowstone/Geyser slot streaming, lifecycle tracking, dynamic tip intelligence, and AI-assisted retry decisions.

## What The Stack Does

- Streams live slot status from Yellowstone/Geyser when `YELLOWSTONE_ENDPOINT` is configured, with RPC slot polling as fallback.
- Detects Jito leader-window context through the Jito block engine.
- Builds Jito bundles with `jito-ts`, a real transaction, and an agent-selected Jito tip account payment.
- Tracks submitted, processed, confirmed, and finalized lifecycle stages with timestamps, slots, and latency deltas.
- Captures network state at submission and confirmation for delta analysis.
- Classifies failures and sends retry context to the AI agent.
- Demonstrates blockhash-expiry recovery through fault injection.
- Supports reactive mainnet operation from Marinade staking events of 1 SOL or more.

## How To Run

```powershell
npm install
copy .env.example .env
npm run build
npm run run:stack
npm run run:inject
npm run generate:evidence
npm run evidence:audit
```

For mainnet evidence, set:

```text
SOLANA_NETWORK=mainnet-beta
SOLANA_RPC_URL=<mainnet rpc>
SOLANA_WS_URL=<mainnet websocket>
YELLOWSTONE_ENDPOINT=<solinfra yellowstone endpoint>
YELLOWSTONE_TOKEN=<solinfra api key/token>
JITO_RPC_URL=https://mainnet.block-engine.jito.wtf/api/v1
```

## Evidence Files

- `logs/lifecycle.json`: raw lifecycle evidence.
- `evidence/run-summary.json`: aggregate statistics.
- `evidence/verification-report.md`: requirement checks and run summary.
- `evidence/blockhash-recovery.md`: focused explanation of the blockhash-expiry recovery scenario.
- `evidence/mainnet-observations.md`: generated by `npm run generate:mainnet-report` after mainnet runs.

## Best Judge Signals To Verify

- Successful records include `signature`, `explorerUrl`, stage timestamps, and non-zero slots.
- Mainnet Jito records include `bundleId` when the block engine accepts the bundle.
- `agentDecisionTrace` shows the exact AI decision family, action, tip, retry, and blockhash-refresh reasoning.
- Fault records include `failure`, `faultInjected`, `retryCount`, and classifier-grounded recovery context.
- `networkSnapshot`, `leaderWindow`, and latency deltas show the stack reacted to observed network conditions.

## AI Agent Responsibility

The agent owns tip intelligence before submission and retry reasoning after failures. The core stack supplies live tip percentiles, slot timing, leader-window context, failure classification, previous tip, retry count, and blockhash-expiry context. The agent returns the operational action and reasoning, while guardrails clamp unsafe tip/wait outputs.

## Known Limitations

- Devnet Jito bundle acceptance is limited because Jito block-engine infrastructure is mainnet-oriented. Devnet runs still produce real Solana transaction lifecycle evidence.
- Accepted `bundleId` evidence is strongest on mainnet with a working Jito endpoint.
- `npm run evidence:audit` is intentionally strict and should fail until the lifecycle log contains fresh mainnet/devnet evidence at the target quality level.
